# 贡献指南

感谢你愿意帮助改进知识星云。提交问题或代码前，请先搜索现有 Issue，避免重复反馈。

## 反馈问题

- Bug 请使用仓库的 Bug 模板，并提供操作系统、Node.js 或 Docker 版本、复现步骤和错误日志。
- 功能建议请说明使用场景、期望行为，以及现有方案不能满足需求的原因。
- 安全漏洞不要创建公开 Issue，请按照 [安全策略](SECURITY.md) 私下报告。

## 本地开发

需要 Node.js 20 或更高版本与 npm。

```bash
npm ci
npm run dev
```

应用的文件读写接口由 `server.mjs` 提供。验证完整生产流程时，请执行：

```bash
npm run build
npm start
```

首次打开后，在设置中选择一个专门用于测试的知识库目录。请勿使用包含重要文件且没有备份的目录进行开发测试。

## 提交改动

1. 每个 Pull Request 只解决一个明确问题。
2. 保持界面文案简洁，并同时检查桌面端和移动端布局。
3. 涉及文件访问、路径处理或 Markdown 渲染时，优先补充安全边界测试。
4. 不要提交真实知识内容、个人绝对路径、凭据、`.env` 或构建产物。
5. 提交前运行全部检查：

```bash
npm test
npm run lint
npm run build
```

Pull Request 描述应包含改动目的、验证方式，以及必要的界面截图或兼容性说明。

# Knowledge Nebula 代码审查与改动说明

## 本次修复

1. 移除源码、界面和部署脚本中的固定知识库目录。
2. 新增网页设置界面，可验证、保存、更换和清除知识库绝对路径。
3. 配置保存在当前用户的应用配置目录，并使用原子写入；配置文件权限按运行平台能力限制。
4. 健康检查、启动日志和通用错误不再暴露完整知识库路径。
5. 未配置或目录暂不可用时给出明确状态，不再回退到个人默认目录。
6. 保留路径越界、符号链接、文件类型、大小和编辑冲突防护。
7. 增加服务端集成测试，覆盖首次未配置、非法相对路径、保存、重启恢复、清除和越界拦截。
8. 增加 ESLint 配置和 GitHub Actions 持续集成。

## 已有能力

- 文件变化通过 SSE 通知浏览器实时刷新。
- Markdown、文本、JSON、配置文件和常见代码文件可直接编辑。
- 保存时使用修改时间进行乐观锁检测，避免覆盖外部新内容。
- 使用同目录临时文件加重命名完成原子保存。
- 图片、PDF 和音视频使用版本参数刷新浏览器缓存。
- 默认仅监听本机回环地址。

## 验证命令

```bash
node --check server.mjs
npm test
npm run lint
npm run build
```

4. 增加 Markdown 双栏编辑与附件管理。

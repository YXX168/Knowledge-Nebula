# Knowledge Nebula · 知识星云

一个本地优先的沉浸式知识库浏览器。它通过仅监听本机的 Node.js 服务读取你选择的目录，提供目录树、全文搜索、文件预览、安全文本编辑和实时同步。

## 主要功能

- 在设置界面配置或更换知识库目录，不在源码中固定任何本机路径
- 文件名、相对路径和正文联合搜索
- 预览 Markdown、文本、代码、JSON、图片、PDF、音频和视频
- 常见文本文件可直接编辑，带外部修改冲突检测和原子保存
- 自动发现目录内新增、修改和删除，通过 SSE 实时刷新页面
- 路径越界和符号链接防护，单个文本文件限制为 2 MB
- 收藏保存在浏览器本地，支持响应式桌面和移动端布局

## 环境要求

- Node.js 20 或更高版本
- npm

## 本地运行

```bash
npm ci
npm run build
npm start
```

打开 `http://127.0.0.1:8765`，首次进入时在设置窗口填写知识库的绝对路径。服务会验证目录存在且可读取后再保存。

Linux/macOS 也可运行：

```bash
bash 启动服务.sh
```

开发模式需要分别运行前端开发服务和本地 API 服务；日常使用建议采用上面的生产构建方式。

## 配置

知识库路径默认保存在当前用户的应用配置目录，不会写入项目或浏览器存储：

- Windows：`%APPDATA%/Knowledge-Nebula/config.json`
- Linux：`$XDG_CONFIG_HOME/knowledge-nebula/config.json`，未设置时使用用户配置目录
- macOS：当前实现使用用户配置目录下的 `.config/knowledge-nebula/config.json`

可用环境变量：

| 变量 | 默认值 | 用途 |
| --- | --- | --- |
| `HOST` | `127.0.0.1` | 服务监听地址 |
| `PORT` | `8765` | 服务端口 |
| `SCAN_INTERVAL_MS` | `1200` | 文件变化扫描间隔，最小 500 ms |
| `KNOWLEDGE_ROOT` | 空 | 未保存配置时的可选初始目录 |
| `KNOWLEDGE_CONFIG_PATH` | 用户配置目录 | 自定义配置文件位置，主要用于便携部署或测试 |

## 安全与隐私

- 默认只监听 `127.0.0.1`，知识文件不会上传到第三方服务。
- 仓库不包含个人知识库路径；实际路径仅保存在运行设备的用户配置文件中。
- 健康检查、启动日志和一般错误不会返回完整知识库路径。
- 不要在没有鉴权和 HTTPS 的情况下把 `HOST` 改为 `0.0.0.0` 暴露到公网或不可信局域网，因为该服务包含文件读取与编辑能力。

Knowledge Nebula 本地部署说明

架构：React + TypeScript + Vite 前端，Node.js 本地服务。
默认地址：http://127.0.0.1:8765
知识库目录：首次打开网页后在设置中填写，可随时更换或清除。

一、部署

1. 安装 Node.js 20 或更高版本。
2. 解压或克隆项目，并进入项目目录。
3. 执行 npm ci。
4. 执行 npm run build。
5. 执行 npm start。
6. 打开 http://127.0.0.1:8765，在设置窗口填写知识库的绝对路径。

Linux/macOS 用户也可以执行：bash 启动服务.sh

Docker Compose 部署：

1. 复制 .env.example 为 .env。
2. 在 .env 中设置 KNOWLEDGE_DIR，或使用默认的项目 knowledge 目录。
3. 执行 docker compose up -d --build。
4. 打开 http://127.0.0.1:8765。

应用配置保存在 Docker 命名卷中，知识库目录以读写方式挂载到容器内的 /knowledge。

二、可选环境变量

HOST=127.0.0.1
PORT=8765
SCAN_INTERVAL_MS=1200
KNOWLEDGE_ROOT=可选的初始绝对路径
KNOWLEDGE_CONFIG_PATH=可选的配置文件绝对路径

通常无需设置 KNOWLEDGE_ROOT。网页保存配置后，路径会放在当前用户的应用配置目录，不会写入项目源码。

三、安全提示

默认只允许本机访问。如果显式设置 HOST=0.0.0.0，文件读取和编辑接口也会对网络开放；请先增加可靠鉴权和 HTTPS，并限制可信来源，切勿直接暴露到公网。

四、功能说明

- 后台自动扫描知识库，新增、修改或删除文件后通知页面刷新。
- 常见文本、Markdown、JSON 和代码文件可安全编辑。
- 编辑期间若文件被其他程序修改，保存会返回冲突提示，不覆盖外部新内容。
- 保存采用同目录临时文件和重命名，降低中断造成文件损坏的风险。
- 禁止访问知识库根目录之外的路径，并跳过符号链接。
- 单个文本文件预览与编辑上限为 2 MB。

五、部署后检查

浏览器或命令行访问：http://127.0.0.1:8765/api/health

返回 ok=true 后即可使用。健康检查只报告是否已配置及是否可用，不会暴露真实知识库路径。

# 安全策略

知识星云能够读取和修改用户指定目录中的文件，因此路径隔离、内容渲染和访问控制相关问题都应按安全问题处理。

## 支持范围

安全修复只面向 `main` 分支的最新版本。旧版本发现问题后，请先升级并确认问题仍然存在。

## 私下报告漏洞

请通过 GitHub 仓库的 [私下漏洞报告](https://github.com/YXX168/Knowledge-Nebula/security/advisories/new) 提交，不要创建公开 Issue。报告中请包含：

- 问题类型和影响范围
- 可复现的最小步骤
- 受影响的版本、操作系统与部署方式
- 必要的日志或截图（请移除知识内容、路径、凭据等敏感信息）
- 如果已经有修复思路，可以一并说明

维护者确认问题并准备好修复前，请避免公开漏洞细节。

## 部署边界

当前版本没有内置用户鉴权，不应直接暴露到公网。需要远程访问时，请在可信反向代理之后配置 HTTPS、身份验证、来源限制和必要的网络防火墙规则，并以最小权限运行服务。
